🪄 Claude superpowers 가이드

Claude Code 의 superpowers 플러그인 설치 + 핵심 스킬 사용법. 스터디 실습 과제를 마친 학생이 본인 개선 아이디어를 직접 기획하고 brainstorming → writing-plans → executing-plans 같은 정형 워크플로로 구현까지 이어갈 수 있도록 안내합니다.

📖 superpowers 란?

superpowers 는 Claude Code 에 플러그인 형태로 설치되는 "스킬(skill)" 모음입니다. 브레인스토밍 · 계획 작성 · 실행 · 검증 · 코드 리뷰처럼 자주 반복되는 작업 워크플로를 미리 정형화해 두어, Claude 가 한 단계도 건너뛰지 않고 일관된 품질로 따라가도록 돕습니다.

예를 들어 /superpowers:brainstorming 을 호출하면 Claude 는 임의로 코드를 짜는 대신 요구사항을 질문으로 풀어 설계 스펙을 먼저 정리하고, 이후 /superpowers:writing-plans 로 task 단위 실행 계획을 만들고, /superpowers:subagent-driven-development 로 각 task 를 분업해서 진행합니다.

이 가이드의 목표: 스터디에서 만든 대시보드/캘린더에 본인만의 기능(예: 댓글 키워드 분석, YouTube 통합, 새로운 통계 뷰) 을 추가하고 싶을 때, superpowers 스킬을 그대로 따라 호출해서 안정적으로 마무리하는 것.
1 Superpowers 플러그인 설치

프로젝트 폴더에서 claude 를 실행한 뒤, 프롬프트에 아래 두 줄을 차례로 입력합니다.

/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace

설치가 끝나면 claude 를 한 번 종료했다가 재시작하세요.

확인:

/help

출력 중 superpowers:* 로 시작하는 스킬들이 보이면 정상입니다. (brainstorming, writing-plans, executing-plans 등)

참고: 마켓플레이스 URL · 슬래시 명령은 Claude Code 버전에 따라 달라질 수 있습니다. /help 또는 공식 GitHub 에서 최신 명령을 확인하세요.
2 핵심 스킬 4개 — 바이브 코딩 레벨

개발자가 아니어도 바로 쓸 수 있는 핵심 4개입니다. 슬래시 명령을 그대로 입력하면 Claude 가 해당 흐름으로 진입합니다.

/superpowers:brainstorming
① 아이디어 정리 — 머릿속 막연한 아이디어를 Claude 와 대화하며 구체적인 설계로 만듭니다. Claude 가 질문 하나씩 던지면서 빠진 부분을 채워줍니다.
이럴 때: "이런 기능 만들고 싶은데 어떻게 시작할지 모르겠어"
예시 prompt: /superpowers:brainstorming 댓글 키워드 분석 기능을 추가하고 싶어
/superpowers:writing-plans
② 만들 순서 정하기 — 정리된 아이디어를 "1단계 → 2단계 → 3단계" 식으로 작은 작업으로 쪼개 계획서를 만듭니다. 한 번에 다 안 만들고 단계별로.
이럴 때: 아이디어는 정해졌는데 어디부터 손대야 할지 막막할 때
예시: brainstorming 직후 자동 연결됨
/superpowers:subagent-driven-development
③ 실제로 만들기 — 계획서를 작은 task 로 나눠 Claude 가 알아서 하나씩 만들고, 각 task 마다 자기 작업을 두 번 검증합니다. 비개발자가 가장 안심하고 맡길 수 있는 흐름.
이럴 때: writing-plans 로 계획서가 준비된 뒤 실제 코드를 만들 차례
대안: 더 간단한 작업이면 /superpowers:executing-plans
/superpowers:systematic-debugging
④ 안 될 때 원인 찾기 — 뭔가 작동 안 하거나 이상할 때 Claude 가 "추측" 대신 "가설 → 확인 → 수정" 순서로 원인을 차근차근 추적합니다.
이럴 때: "왜 안 되는지 모르겠는데 자꾸 엉뚱한 수정만 함" 같은 상황
예시 prompt: /superpowers:systematic-debugging 댓글 panel 이 새로고침 시 비어 보임

📌 그 외 개발자용 스킬 (test-driven-development, using-git-worktrees, requesting-code-review, writing-skills 등) 은 코드 깊게 만지기 시작할 때 차츰 활용. 비개발자가 처음부터 다 알 필요는 없습니다.

3 실습 시나리오 3개

아래는 본 프로젝트(대시보드/캘린더)에 그대로 적용해볼 수 있는 예시 흐름입니다. 각 슬래시 명령을 차례로 입력하면 됩니다.

(가) 팔로워 댓글 키워드 분석 기능 추가 기능 추가

view-customers 에 인스타 댓글의 키워드 빈도 분석을 붙이고 싶다면.
1. /superpowers:brainstorming
   "view-customers 에 인스타 댓글의 키워드 빈도 분석 기능을
   추가하고 싶어. 어떤 단어가 자주 나오는지 워드클라우드로 보여주고 싶음."

2. /superpowers:writing-plans
   (브레인스토밍에서 합의한 스펙 기반으로 task 계획서 작성)

3. /superpowers:subagent-driven-development
   (계획서의 task 를 하나씩 분업 실행 + review)

(나) 새 SNS 플랫폼 통합 — YouTube 통합

dashboard 에 YouTube 통계 view 를 추가하고, 핸들 입력 후 자동 호출.
1. /superpowers:brainstorming
   "dashboard 에 YouTube 통계 view 를 추가하고 싶어.
   YouTube Data API v3 사용. 핸들 입력하면 구독자/조회수/최근 영상 표시."

2. /superpowers:writing-plans
   (API 키 입력 UI, fetch 로직, 카드 렌더, localStorage 캐시 — task 분해)

3. /superpowers:executing-plans
   (사람 체크포인트 거치며 단계별 실행)

(다) 버그 추적 디버깅

재현이 불안정해서 원인이 모호한 경우.
1. /superpowers:systematic-debugging
   "fetchProducts 호출 후 가끔 빈 테이블이 떠.
   네트워크 탭에서 응답은 정상으로 보이는데 원인 모르겠음."

   → Claude 가 가설을 세우고, 로그/재현 명령으로 검증한 뒤
     실제 원인을 좁혀가며 수정 PR 까지 제안.
4 권장 흐름 & 더 알아보기

큰 작업은 아래 순서를 그대로 따라가는 것이 가장 안전합니다.

brainstorm → write plan → review plan → execute → verify
  • 각 단계 끝마다 본인이 직접 결과를 읽어보고, 마음에 들지 않으면 같은 스킬을 다시 호출.
  • execute 중간에 막히면 /superpowers:systematic-debugging 으로 분기.
  • 완료 직전엔 반드시 /superpowers:verification-before-completion.
