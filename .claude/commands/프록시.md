Cloudflare Pages Functions 기반 CORS 프록시를 생성/수정합니다.

## 파일
- `functions/api/[[path]].js` — 리버스 프록시 (catch-all)

## 동작 원리
브라우저에서 외부 API를 직접 호출하면 CORS 정책으로 차단됨.
Cloudflare Pages Function이 같은 도메인에서 요청을 받아 서버 측에서 대상 API로 전달.
서버 간 통신에는 CORS가 적용되지 않으므로 우회 가능.

```
브라우저 (<your-project>.pages.dev)
  → /api/admin/v2/... (같은 도메인, CORS 없음)
  → Cloudflare Pages Function (서버)
    → api.store.friendly-pharmacist.com (서버↔서버, CORS 없음)
  ← 응답 전달
```

## 프록시 구조
- `onRequest(context)` — 모든 HTTP 메서드 처리
- OPTIONS preflight → 204 + CORS 허용 헤더 반환
- GET/POST/PUT/DELETE → 대상 URL로 요청 전달, 응답에 `Access-Control-Allow-Origin: *` 추가
- `host`, `origin`, `referer` 헤더 제거 후 전달

## 대시보드 연동
- 배포 환경: `API_BASE = ''` → `/api/...` 상대경로로 프록시 경유
- 로컬 환경: `API_BASE = 'https://api.store.friendly-pharmacist.com'` → 직접 호출

## 수정 시 주의
- 프록시 대상 URL(`TARGET` 상수)은 프록시 파일 내에서만 관리
- 인증 헤더(Authorization)는 클라이언트에서 설정한 그대로 전달됨
- `functions/` 디렉토리 구조가 Cloudflare Pages 라우팅 규칙을 따름
- 새 API 대상 추가 시 별도 프록시 파일 또는 TARGET 분기 고려
- 수정 완료 후 `/deploy` 명령으로 배포 안내
